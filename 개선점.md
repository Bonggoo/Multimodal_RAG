# 프로젝트 개선점 및 향후 과제

## 완료된 개선 사항 (2025-12-11)

### 1. 검색 성능 (Retrieval) - **완료**
- **현상 해결**: "원점 복귀 방식 종류"와 같은 구체적 개념 질문에 대해 정확한 답변을 생성하도록 개선함.
- **적용된 기술**:
    1.  **하이브리드 검색 (Hybrid Search)**: `EnsembleRetriever`를 도입하여 `BM25Retriever`(키워드)와 `VectorStoreRetriever`(의미)를 결합 (가중치 50:50).
    2.  **쿼리 확장 (Query Expansion)**: LLM을 이용해 사용자 질문을 검색 친화적 키워드(동의어, 기술 용어 포함)로 확장하는 전처리 단계 추가.
    3.  **메타데이터 고도화**: 문서 파싱 시 `keywords`(핵심 용어)와 `summary`(요약)를 추출하여 메타데이터에 포함, 검색 정확도 향상.

### 2. 문서 파싱 (Parsing) - **완료**
- **안정성 강화**: `gemini-2.5-flash` 모델의 JSON 파싱 실패 시, 지수 백오프(Exponential Backoff)를 적용한 재시도(Retry) 로직(최대 3회)을 구현하여 파이프라인의 견고함 확보.

### 3. 사용자 경험 (UX) - **완료**
- **진행 상황 시각화**: `tqdm` 라이브러리를 적용하여 문서 파싱 및 적재 과정의 진행률과 예상 소요 시간을 직관적으로 표시.
- **최적화**: PDF 파일 로딩 방식을 개선하여 불필요한 파일 I/O 오버헤드 감소.

## 향후 과제 (Backlog)

### 1. 멀티모달 검색 고도화
- 현재는 텍스트/테이블/이미지 설명 텍스트를 기반으로 검색하지만, **이미지 자체 임베딩(Multi-modal Embeddings)**을 활용하여 이미지 쿼리로 이미지를 찾는 기능 검토.

### 2. 답변 품질 평가 (Evaluation)
- RAGAS 등의 프레임워크를 도입하여 답변의 정확성(Faithfulness), 관련성(Answer Relevance)을 정량적으로 평가하는 파이프라인 구축.
