## 6. JOG 운전 시 속도 제한 수치를 설정하는 파라미터는 무엇인가요?

**지연 시간:** 12.20s

### 답변:
JOG 운전 시 속도 제한 수치를 설정하는 파라미터는 **Pr.31 JOG 속도 제한값**입니다. 이 파라미터는 JOG 운전 시의 속도 제한값(제어 시의 최고 속도)을 설정하며, **Pr.8 속도 제한값** 이하로 설정해야 합니다. 만약 Pr.31이 Pr.8보다 큰 값을 설정하면 축 에러(에러 코드: 956)가 발생하여 기동하지 않습니다.

또한, Cd.17 JOG 속도 설정값이 Pr.31 JOG 속도 제한값에 설정된 속도를 초과하는 경우, Pr.31 JOG 속도 제한값으로 동작하며 QD77MS는 축 경고(경고 코드: 301)가 발생합니다.

## 9. 위치결정 제어 중 'ABS(절대값)'와 'INC(증분값)' 시스템의 결정적인 차이는?

**지연 시간:** 5.72s

### 답변:
위치결정 제어에서 ABS(절대값) 시스템과 INC(증분값) 시스템의 결정적인 차이는 다음과 같습니다.

*   **ABS(절대값) 방식**: 현재 위치를 원점으로부터의 절대적인 주소로 설정합니다. 즉, 항상 원점을 기준으로 현재 위치를 파악합니다.
    *   속도·위치 전환 제어(ABS 모드) 시에는 속도 제어에서 위치 제어로 전환된 후의 목표값이 되는 절대 어드레스를 설정합니다. 이 경우 단위는 'degree'만 유효합니다.
    *   [Image Source: assets/images/QD77MS_위치결정_매뉴얼/page_199.png]
    *   [Image Source: assets/images/QD77MS_위치결정_매뉴얼/page_499.png]

*   **INC(증분값) 방식**: 현재 위치로부터의 이동량을 설정합니다. 즉, 현재 위치에서 얼마나 이동할지를 지정합니다.
    *   속도·위치 전환 제어(INC 모드) 시에는 속도 제어에서 위치 제어로 전환된 후의 이동량을 설정합니다.
    *   [Image Source: assets/images/QD77MS_위치결정_매뉴얼/page_199.png]
    *   [Image Source: assets/images/QD77MS_위치결정_매뉴얼/page_499.png]

또한, ABS 모드에서 속도·위치 전환 제어를 실행하기 위해서는 `Pr.1 단위 설정`이 'degree'로 설정되어야 합니다.

## 10. 티칭(Teaching) 기능을 사용하여 목표 위치를 저장하는 절차는?

**지연 시간:** 6.10s

### 답변:
티칭 기능을 사용하여 목표 위치를 저장하는 절차는 다음과 같습니다.

1.  **기계 원점 복귀 실행**: 먼저 대상 축의 기계 원점 복귀를 실행합니다.
2.  **수동 운전으로 목표 위치 이동**: JOG 운전, 인칭 운전, 또는 수동펄서 운전을 사용하여 원하는 목표 위치까지 이동시킵니다.
3.  **티칭 데이터 선택 설정**:
    *   "Da.6 위치결정 어드레스/이동량"에 기입하는 경우: 티칭 데이터 선택에서 "[이송 현재값을 "Da.6 위치결정 어드레스/이동량"에 기입한다]"를 설정합니다.
    *   "Da.7 원호 어드레스"에 기입하는 경우: 티칭 데이터 선택에서 "[이송 현재값을 "Da.7 원호 어드레스"에 기입한다]"를 설정합니다.
4.  **버퍼 메모리 설정**:
    *   "Da.6 위치결정 어드레스/이동량"에 기입하는 경우: 버퍼 메모리 주소 [1548]에 0을 설정합니다.
    *   "Da.7 원호 어드레스"에 기입하는 경우: 버퍼 메모리 주소 [1548]에 1을 설정합니다.
5.  **위치결정 데이터 No. 설정**: 티칭을 실행할 위치결정 데이터 번호를 버퍼 메모리 주소 [1549]에 설정합니다.
6.  **티칭 실행**: 전용 명령 `ZP.TEACH1` (축 1의 경우) 등을 사용하여 티칭을 실행합니다.
7.  **티칭 완료 확인**: 버퍼 메모리 주소 [1549]가 0이 되었는지 확인하여 티칭 완료를 확인합니다.
8.  **PLC 레디 신호 OFF**: PLC 레디 신호 [YO]를 OFF 합니다.
9.  **플래시 ROM 기입 요구**: 플래시 ROM으로의 기입 요구를 위해 버퍼 메모리 주소 [1900]에 1을 설정합니다.
10. **기입 완료 확인**: 버퍼 메모리 주소 [1900]이 0이 되었는지 확인하여 기입 완료를 확인합니다.

이 과정은 수동 제어로 위치 맞춤한 어드레스를 위치결정 데이터의 어드레스에 설정하는 것을 목표로 합니다.

## 11. 2축 직선 보간 제어 시 합성 속도 계산 방식은?

**지연 시간:** 4.70s

### 답변:
2축 직선 보간 제어 시 합성 속도 계산 방식은 다음과 같습니다.

Pr.20 보간속도 지정방법에서 '0: 합성속도'를 설정하면, 제어 대상의 이동 속도를 지정하고 각 축의 속도는 QD77MS가 계산합니다.

하지만, 4축 직선 보간, 2축~4축의 속도 제어 시에는 "기준축 속도"를 지정해야 합니다. 만약 이 경우 "합성속도"를 지정하고 위치결정을 기동하면 "보간모드 에러" (에러코드: 523)가 발생합니다.

또한, 원호 보간 제어를 실행할 경우에는 "합성속도"를 지정해야 합니다.

## 12. 인터럽트 위치결정 제어 기동 시 필요한 외부 입력 신호는?

**지연 시간:** 3.54s

### 답변:
인터럽트 위치결정 제어 기동 시 필요한 외부 입력 신호는 다음과 같습니다.

*   **외부 지령 신호**: 위치결정 제어를 외부에서 직접 기동하기 위해 사용됩니다. 이 신호를 사용하면 PLC CPU의 1스캔타임 분의 변화 시간을 없앨 수 있어, 기동 지령을 조금이라도 빨리 내리거나 기동의 변화 시간을 억제하고 싶을 때 유용합니다. 외부 지령 신호를 사용하기 위해서는 파라미터(Pr.42) 설정을 통해 외부 지령 기능을 선택하고, 프로그램 No.5 "외부 지령 신호 유효 설정 프로그램"에서 외부 지령 신호를 유효하게 설정해야 합니다. QD77MS16 사용 시에는 Pr.95 "외부지령 신호 선택"에서 사용하는 외부 지령 신호를 설정해야 합니다.
*   **위치결정 기동 신호**: 위치결정 제어를 시작하기 위한 신호입니다.
*   **전체축 서보ON 신호**: 모든 축의 서보를 ON 상태로 만드는 신호입니다.
*   **PLC 레디 신호**: PLC가 준비 완료 상태임을 나타내는 신호입니다.
*   **QD77 준비완료 신호**: QD77MS 모듈이 준비 완료 상태임을 나타내는 신호입니다.

이 외에도 기동 완료 신호, BUSY 신호, 위치결정 완료 신호, 에러 검출 신호 등이 동작 상태를 나타내거나 제어에 관여합니다.

## 13. 파라미터 초기화(Flash ROM) 시 주의사항은 무엇인가요?

**지연 시간:** 5.08s

### 답변:
파라미터 초기화(Flash ROM) 시 주의사항은 다음과 같습니다.

*   **위치결정 제어 미실행:** 파라미터 초기화는 위치결정 제어를 실행하지 않을 때(PLC 레디 신호[Y0]가 OFF 시) 실행해야 합니다. PLC 레디 신호[Y0]가 ON일 때 실행하면 경고 코드 111이 발생합니다.
*   **플래시 ROM 기입 횟수 제한:** 플래시 ROM의 기입 횟수는 10만회입니다. 10만회를 초과하면 플래시 ROM으로의 기입이 불가능해지며, 플래시 ROM 기입 에러(에러 코드 801)가 발생합니다.
*   **PLC CPU 리셋 또는 전원 재기동 필요:** 파라미터 초기화를 실행한 후에는 PLC CPU의 리셋 또는 PLC CPU의 전원 재기동을 수행해야 합니다.
*   **처리 시간:** 파라미터 초기화 처리 시간은 약 10초이며, 최대 30초 정도 소요될 수 있습니다.
*   **처리 중 전원 ON/OFF 및 리셋 금지:** 파라미터 초기화 중에는 전원의 ON/OFF 또는 PLC CPU의 리셋을 수행하지 않아야 합니다. 플래시 ROM/보존용 내부 메모리로 기입 중에 전원이 꺼지거나 PLC CPU 유닛이 리셋되어 처리가 강제로 중단되면, 백업된 데이터가 소실될 수 있습니다.

## 14. 에러 리셋(Error Reset)을 수행하는 버퍼 메모리 주소를 알려줘.

**지연 시간:** 3.58s

### 답변:
에러 리셋을 수행하는 버퍼 메모리 주소는 다음과 같습니다.

*   **에러 리셋 지령:** `X3E` (1473)
*   **에러 리셋 클리어 지령:** `X4C` (1484)

에러 리셋 요구 시 서보 에러 리셋이 불가능한 경우, 축 에러 리셋 값은 QD77MS에 의해 "0"이 아닌 "1"로 유지됩니다. 재차 에러 리셋을 실행하려면, 먼저 축 에러 리셋에 "0"을 설정한 후 "1"을 설정해야 합니다.
